# 代码审查检查项模板

## 通用检查项（所有语言）

### 功能正确性
- [ ] 代码实现了需求描述的功能，没有遗漏
- [ ] 边界条件已处理（空值、空列表、最大值、最小值）
- [ ] 并发场景已考虑（竞态条件、重复提交、幂等性）

### 日志与可观测性
- [ ] 核心业务流程有 INFO 日志，包含关键上下文参数
- [ ] 异常流程有 ERROR 日志，包含错误详情和触发上下文
- [ ] HTTP/RPC 请求有输入输出日志，包含耗时
- [ ] 日志消息使用英文，不记录密码/Token 等敏感信息

### 错误处理
- [ ] 没有空 catch/except（吞掉异常）
- [ ] 错误信息有上下文，不只是 "error occurred"
- [ ] 对外接口的错误返回了正确的错误码和描述
- [ ] 数据库/外部服务调用有超时设置

### 安全
- [ ] 用户输入有校验，没有直接拼接到 SQL/命令行
- [ ] 敏感数据（密码、Token、手机号）没有明文记录到日志
- [ ] API 接口有鉴权，不存在未授权访问
- [ ] 文件上传有类型和大小限制

### 代码质量
- [ ] 函数/方法的职责单一，不超过 50 行（特殊情况除外）
- [ ] 没有魔法数字/字符串（应定义为常量或枚举）
- [ ] 命名准确反映其含义，不使用 a/b/tmp/data 等模糊名称
- [ ] 没有注释掉的废弃代码（直接删除）
- [ ] 没有 TODO/FIXME（应创建 issue 跟踪）

### 测试
- [ ] 核心逻辑有单元测试
- [ ] 测试覆盖正常路径和异常路径
- [ ] 测试不依赖外部状态（网络、文件系统、时间）

## Python 特定检查项

- [ ] 数据结构使用 pydantic BaseModel，不用裸 dict
- [ ] 没有 getattr/setattr（非特殊情况）
- [ ] 函数有 type hints
- [ ] 异步函数（async def）中没有使用 requests（应用 httpx）

## Go 特定检查项

- [ ] 数据结构使用 struct，不用 map[string]interface{}
- [ ] 所有 error 已处理，没有 `_ = err`
- [ ] 错误信息有 %w 包装，保留调用栈
- [ ] 函数第一个参数是 ctx context.Context（如有上下文传递需求）

## TypeScript 特定检查项

- [ ] 没有使用 any 类型
- [ ] 外部输入使用 zod 做运行时校验
- [ ] 没有类型断言（as Type）绕过类型检查

## 部署/配置相关

- [ ] 新增的环境变量已更新 `.env.example`
- [ ] Dockerfile 如有变更，`mkdir -p logs` 仍然存在
- [ ] 没有将密码/API Key 硬编码在代码中
