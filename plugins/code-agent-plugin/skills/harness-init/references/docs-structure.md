## docs/design-docs/core-beliefs.md

# 核心原则

> 这些原则指导所有架构和技术决策。

## 1. 仓库是记录系统
所有架构决策、产品规格、技术债务必须版本化进仓库。
存在于 Slack/Notion/人脑中的知识对 AI 不可见，等同于不存在。

## 2. 约束优于规范
用 linter 强制执行架构规则，而非依赖约定或文档。
规则一旦编码，立即对所有 AI 运行生效。

## 3. 可读性优先
代码和文档首先为 AI 可读性优化，其次为人类。
模块边界清晰、依赖方向明确，优先于代码"优雅"。

## 4. 渐进式披露
AGENTS.md 是地图，不是百科全书。
智能体从小切入点出发，被引导深入，而不是一开始被淹没。
